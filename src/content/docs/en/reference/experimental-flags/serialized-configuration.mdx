This feature allows to use a new virtual module called `astro:config`. The virtual module exposes a non-exhaustive, serializable, type-safe version of the Astro configuration.

The virtual module exposes two "submodules".

## `astro:config/client`

It exposes information that are **safe** to expose to the client (browser), hence it can be used in any instance of the application. It exposes information such as `trailingSlash`, `build.format`, `i18n`, and more.

## `astro:config/server`

It exposes all information exposed by `astro:config/client`, plus information regarding the file system configuration, such as `srcDir`, `cacheDir`, `outDir`, and more. This information **isn't safe** to expose to the client (browsers), 
so attempting to use them wil raise an error.
